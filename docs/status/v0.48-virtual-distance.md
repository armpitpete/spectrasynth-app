# v0.48 — Virtual Distance

Scope: one safe sound feature.

Goal: make Virtual Distance affect the sound without changing the Web Audio graph.

Implemented behaviour:

- Virtual Distance is now active.
- Near values keep the sound louder, brighter, and more present.
- Far values make the sound quieter, darker, and softer.
- The control works by driving the existing Output and Cutoff / Brightness sliders.
- Existing input events are dispatched so the existing safe update functions perform the audio changes.

Mapping:

```text
Virtual Distance 0   → Output 70, Cutoff 63
Virtual Distance 100 → Output 32, Cutoff 34
```

Preserved:

- no gain constants changed
- no new audio nodes added
- no Web Audio graph changes
- no Delay implementation
- no Reverb implementation
- no Two-Moon Movement implementation
- no Band 5 audition
- no feedback
- no filter-bank behaviour
- no deployment workflow changes

Test checklist:

- app loads
- oscillator starts and stops
- noise starts and stops
- Virtual Distance changes sound
- Output still works
- Cutoff still works
- Panic Stop still works
- no console errors
