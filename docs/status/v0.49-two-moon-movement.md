# v0.49 — Two-Moon Movement

Scope: one safe movement feature.

Goal: make Two-Moon Movement affect the sound without changing the Web Audio graph.

Implemented behaviour:

- Two-Moon Movement is now active.
- 0 means no automatic movement.
- Higher values add gentle slow brightness movement.
- The movement is made from two overlapping slow cycles.
- The control works by gently moving the existing Cutoff / Brightness slider.
- Existing input events are dispatched so the existing safe update functions perform the audio changes.

Preserved:

- no gain constants changed
- no new audio nodes added
- no Web Audio graph changes
- no Delay implementation
- no Reverb implementation
- no Band 5 audition
- no feedback
- no filter-bank behaviour
- no deployment workflow changes

Test checklist:

- app loads
- oscillator starts and stops
- noise starts and stops
- Two-Moon Movement changes brightness slowly when above 0
- Cutoff still works
- Virtual Distance still works
- Output still works
- Panic Stop still works
- no console errors
