# v0.76 — Core regression

Manual test after PR #121 showed a core synth regression:

- Cutoff stopped working.
- Resonance stopped working.
- Buttery Fuzz stopped working.
- Oscillator and Noise became too quiet.
- Spectral bands 7–10 did not give useful distinct behaviour.

Interpretation:

The Spectral Engine audio hook became too invasive. It interfered with the normal source-to-tone-filter-to-fuzz path. The recovery step is to restore the proven core audio path first by pausing the v0.76 Spectral Engine audio hook.

Recovery priority:

1. Restore Oscillator and Noise level.
2. Restore Cutoff, Resonance, and Buttery Fuzz.
3. Keep the Spectral Engine UI visible.
4. Do not attempt another full routing experiment inside v0.76.
5. Plan the real 10-band engine separately after the core synth is stable again.

Expected behaviour after this recovery:

- Oscillator works at normal level.
- Noise works at normal level.
- Cutoff changes tone again.
- Resonance changes tone again.
- Buttery Fuzz changes tone again.
- Spectral Engine faders and Mute/Unmute are UI-only until the proper filter-bank engine is designed.
