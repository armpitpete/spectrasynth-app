# v0.76 — Spectral routing recovery target

Manual test after PR #119 showed two remaining faults:

- When all Spectral Engine bands are muted, sound still passes.
- Bands 6–10 behave mostly like volume changes instead of distinct higher spectral bands.

Root cause:

- The previous recovery path still included a dry bypass.
- The spectral band outputs still fed back into the normal low-pass tone filter path, so high bands lost their identity.

Recovery target:

- Remove the dry bypass from the Spectral Engine test path.
- Do not send spectral band outputs back into the normal low-pass tone filter.
- Route spectral band outputs to the downstream audio path that already follows the tone filter.
- All bands muted should silence the Spectral Engine contribution.
- Bands 6–10 should produce clearly different high-frequency colour, not just volume change.

Scope remains v0.76 recovery only. Do not add new features until this passes manual test.
