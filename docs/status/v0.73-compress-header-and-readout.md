# v0.73 — Compress header and Source Readout

Scope: visual/layout CSS only.

Problem:

The header and Source Readout still used too much vertical space after v0.72. The top of the app needed to read more like a compact instrument header/status strip.

Implemented:

- Reduced app top padding.
- Reduced header padding and bottom spacing.
- Reduced SpectraSynth title size.
- Reduced eyebrow and subtitle size/spacing.
- Reduced version pill size.
- Moved the Public Build strip higher to match the smaller header.
- Reduced Source Readout panel padding.
- Reduced Source Readout section-heading spacing.
- Reduced Source Readout card height and padding.
- Reduced Source Readout label/value text sizing.
- Kept all Source Readout fields visible and readable.

Preserved:

- no audio changes
- no Source Readout behaviour changes
- no Scale Chance behaviour changes
- no AR envelope changes
- no arp or cluster behaviour changes
- no Spectral Engine behaviour changes
- no oscillator pitch changes
- no MIDI
- no Delay, Reverb, or Stereo Width changes
- no deployment workflow changes

Test:

- app loads
- header is much shorter
- Source Readout is much thinner
- all readout values remain readable
- Spectral Engine remains directly below Source Readout
- Source / Movement / Effects row remains below Spectral Engine
- existing controls still work
