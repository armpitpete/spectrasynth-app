# v0.65 — Arp Clusters drive Cutoff movement

Scope: Scale Chance / Cutoff movement only.

Implemented:

- Cluster Mode now affects the Scale Chance cutoff engine.
- When Cluster Mode is On, the engine uses Cluster Count and Cluster Size.
- The engine steps through the current cluster as a small arp.
- When a cluster finishes, the engine advances to the next active cluster.
- Each selected cluster note is converted to frequency and mapped to the existing Cutoff / Brightness slider.
- Existing Note Length and Note Gap timing is used.
- Rest Chance remains active.
- Repeat Chance remains active.

Cluster Direction:

- As Selected follows selected note order inside each cluster.
- Up sorts notes low to high.
- Down sorts notes high to low.
- Up / Down rises then falls without repeating end notes twice inside the phrase.
- Random chooses one note from the current cluster each cycle.

Preserved:

- no oscillator pitch automation
- no new audio nodes
- no MIDI
- no sequencer grid
- no polyphony
- no presets
- no pattern saving
- no filter-bank behaviour
- no Band 5 audition
- no feedback loop
- no Delay, Reverb, or Stereo Width changes
- no gain safety increase
- no deployment workflow changes

Test:

- app loads
- Cluster Mode Off leaves existing chance and arp behaviour unchanged
- Cluster Mode On moves Cutoff through selected cluster notes
- Cluster Count changes how many clusters are used
- Cluster Size changes how many notes are used inside each cluster
- Cluster Direction changes note order inside clusters
- Rest Chance still skips some cycles
- Repeat Chance still repeats some targets
- existing synth controls still work
