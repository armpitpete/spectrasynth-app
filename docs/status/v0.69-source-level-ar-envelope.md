# v0.69 — Source-level Scale Chance AR envelope

Scope: AR envelope routing correction only.

Problem:

v0.68 used the main Output slider as the AR envelope target. That made the envelope act like a master-volume gate and chopped everything downstream, including delay/reverb/stereo tails.

Corrected model:

```text
AR envelope → source level → cutoff/filter → fuzz → delay/reverb/stereo → Output
```

Implemented:

- AR Envelope no longer changes the Output slider.
- Output remains a normal master level control.
- AR Envelope now tracks oscillator/noise source gain stages.
- Scale Chance note events now trigger source-level attack/release articulation.
- Rest cycles still do not trigger the envelope.
- Turning AR Envelope On silences tracked source gains until the next note event.
- Turning AR Envelope Off restores tracked source gains to normal quiet source levels.
- Panic Stop still cancels and silences the AR envelope safely.
- Delay/reverb/stereo tails are no longer intentionally chopped by the AR envelope.

Preserved:

- no oscillator pitch automation
- no new sequencer grid
- no MIDI
- no polyphony
- no presets
- no pattern saving
- no filter-bank behaviour
- no Band 5 audition
- no feedback loop
- no Delay changes
- no Reverb changes
- no Stereo Width changes
- no deployment workflow changes

Test:

- app loads
- Output slider stays where the user set it during AR envelope events
- AR Envelope On shapes Scale Chance notes at source level
- Attack changes note onset
- Release changes note falloff
- Delay/reverb/stereo tails are not cut off by master Output gating
- AR Envelope Off restores normal continuous source behaviour
- Panic Stop still silences safely
