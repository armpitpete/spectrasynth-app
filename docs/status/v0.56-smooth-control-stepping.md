# v0.56 — Smooth Cutoff and movement control stepping

Scope: control smoothing only.

Goal: reduce audible stepping / zipper noise when Cutoff or automatic movement changes the sound.

Problem reported:

- Cutoff motion sounded stepped.
- Upward moving sounds did not slide smoothly.

Implemented changes:

- Added `src/smooth-controls.js`.
- Cutoff / Brightness now uses `step="0.1"` instead of whole-number slider movement.
- Two-Moon Movement now writes fractional Cutoff values instead of rounded whole numbers.
- Virtual Distance now maps Cutoff with fractional values instead of rounded whole numbers.

Preserved:

- no new features
- no Delay changes
- no Reverb changes
- no Stereo Width changes
- no feedback loop
- no Band 5 audition
- no filter-bank behaviour
- no gain safety increase
- no deployment workflow changes

Test checklist:

- app loads
- Cutoff sweeps sound smoother
- Two-Moon Movement sounds less stepped
- Virtual Distance still works
- Delay still works
- Reverb still works
- Stereo Width still works
- Panic Stop still works
- no console errors
