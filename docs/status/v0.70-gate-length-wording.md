# v0.70 — Gate Length wording and AR timing clarity

Scope: UI and wording clarity only.

Problem:

After adding the Scale Chance AR Envelope, the old label **Note Length** became misleading. The control is not the full audible note duration. It is the gate time before the envelope release begins.

Correct timing model:

```text
Scale Chance note event
→ Attack rises
→ Gate Length holds the event open
→ Release falls away
→ Note Gap before the next event
```

The approximate shaped event time is:

```text
Attack + Gate Length + Release
```

Implemented:

- Renamed visible Note Length label to Gate Length.
- Kept the existing internal control id to avoid touching engine logic.
- Updated Plain Patch Summary wording from note length to gate length.
- Updated AR Envelope summary wording so Attack, Gate Length, Release, and Note Gap are clearer.
- Kept the source-level AR wording from v0.69.

Preserved:

- no timing behaviour changes
- no Scale Chance engine logic changes
- no AR envelope algorithm changes
- no source routing changes
- no oscillator pitch automation
- no new audio nodes
- no MIDI
- no sequencer grid
- no filter-bank behaviour
- no Delay changes
- no Reverb changes
- no Stereo Width changes
- no deployment workflow changes

Test:

- Scale Chance behaviour is unchanged
