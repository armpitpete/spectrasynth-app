# v0.66 — Collapse Arp Clusters to selected cluster view

Scope: UI compression only.

Problem:

The Arp Clusters feature worked, but the panel showed every cluster and every note selector at once. This made the Scale Chance panel too tall.

Implemented:

- Added Cluster View drop-down.
- Cluster View only lists clusters allowed by Cluster Count.
- Only the selected cluster is shown.
- Only note selectors allowed by Cluster Size are shown.
- Hidden cluster and note selectors remain in the DOM, so the existing engine can still read their values.
- Plain Patch Summary now reports the selected cluster view instead of listing every cluster.

Preserved:

- no Scale Chance engine changes
- no existing arp behaviour changes
- no Cutoff movement changes
- no oscillator pitch automation
- no new audio nodes
- no MIDI
- no sequencer grid
- no polyphony
- no presets
- no pattern saving
- no filter-bank behaviour
- no Band 5 audition
- no feedback loop
- no Delay, Reverb, or Stereo Width changes
- no deployment workflow changes

Test:

- app loads
- Arp Clusters section is much shorter
- Cluster View is a drop-down
- Cluster Count changes the available Cluster View options
- Cluster Size changes how many note controls are visible
- hidden cluster values still play when Cluster Mode is On
- existing Scale Chance / Arp / Cluster behaviour still works
